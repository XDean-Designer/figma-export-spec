导出前设计稿调整指南

本页用于在 Figma 设计稿通过插件导出 HTML 之前，系统化检查并修正设计结构。 插件只能忠实转换稿中已有的信息——设计侧约束到位，还原度才能接近「像素级一致」。

适用：静态页面 / 原型 HTML 目标：误差 ≤ 2px 不承诺绝对 100% 移动端 390px 基准

0目标与边界

合理预期 在固定视口、Web 可用字体已加载、静态页面、无复杂动效的前提下，视觉还原可控制在 1–2px 误差 内。 「100% 还原」应理解为可验收的高保真，而非跨浏览器、跨字体、跨平台的绝对一致。
插件能较好还原
  • Auto Layout → Flexbox 布局
  • 整数 px 的 padding / gap / 圆角
  • 纯色填充、1px 描边
  • 标准文本样式（字号、行高、字重、颜色）
  • SVG 矢量图标
  • Component 实例的结构映射
难以或无法等价还原
  • 绝对定位手工摆放的图层
  • 混合模式、复杂模糊、多重阴影
  • 非 Web 字体（未提供 fallback）
  • 小数 px、非 4 模数间距
  • 文字转曲 / 图片化文字
  • 负间距层叠、复杂蒙版裁切
约束级别 含义 不遵守的后果
必须 导出前必须满足，否则大概率明显偏差 布局错位、间距漂移、文本换行异常
应当 强烈建议，显著影响还原稳定性 偶发 2–8px 偏差，需人工修 HTML
可选 优化项，便于后续维护与组件映射 不影响单次导出，但增加返工成本

1导出前流程

建议按以下顺序操作，避免「导出后再改稿」的反复。

① 结构整理Auto Layout、层级扁平、删除隐藏层
② Token 对齐颜色/字号/间距改用变量或规范值
③ 组件替换散件改为 Component 实例
④ 本页清单逐项勾选，未通过则回到 Figma 修改
⑤ 试导出导出 1 页做截图 diff
⑥ 批量导出通过后批量处理同类页面

3布局与结构

核心原则：Auto Layout 优先 所有 Frame（容器）应启用自动布局。HTML 的 Flexbox 与 Figma Auto Layout 语义最接近；绝对定位是还原度最大的敌人。

3.1 容器规则

对象要求级别
根画板 / 页面 Frame固定宽度（如 390px），纵向 Auto Layout，子元素自上而下排列必须
卡片、列表行、按钮、导航栏独立 Frame + Auto Layout，禁止 Group 代替 Frame必须
嵌套层级语义清晰，建议 ≤ 6 层；过深嵌套会导致导出 DOM 冗余应当
绝对定位仅用于角标、浮层等少量场景；主体内容禁止手工拖拽定位必须
Constraints在 Auto Layout 内优先用 Fill / Hug；少用 Left+Top 固定应当
❌ 错误：Group + 绝对定位
元素靠 eyeball 对齐，导出后 margin/padding 无法稳定复现，换行即错位。
✓ 正确：Frame + Auto Layout

副标题

3.2 画板基准（移动端）

  • 必须 画布宽 390px（1x），高默认 844px，内容超出可向下延展
  • 必须 画板为直角矩形，禁止设备外壳、圆角画板、外描边装饰
  • 必须 状态栏占位 390×44px 纯白块，不绘制系统 UI
  • 必须 底部安全占位 390×39px 纯白块，不绘制 Home Indicator
  • 必须 内容区禁止横向溢出 390px，禁止出现横向滚动

4尺寸与间距

4px 模数 所有 padding、gap、margin、宽高尽量使用 4 的整数倍。插件对整数 px 映射最稳定；小数 px 会在浏览器中四舍五入，产生累计误差。
Token值用途
页面左右边距16px主内容区安全边距
模块间距16px卡片与卡片、区块与区块
卡片内边距16px四向一致或明确标注
图文间距8px图标与文字、标签与文字
列表行间距12px列表项上下 padding
标签内边距4px 8px小标签、角标

圆角规范

场景圆角
全局卡片12px
输入框 / 小模块8px
标签 / 角标4px
按钮仅在 4 / 8 / 12 / 16px 四档中选择，同类型全站统一
圆形使用 999px 或宽高相等 + 50%（插件常映射为 border-radius: 999px）

固定高度组件

  • 标题栏 / 导航栏：44px
  • 主按钮 / 底栏按钮：48px，最大宽度 370px
  • 搜索框：36px
  • 通用图标画布：24×24px（可视区 20×20）

文本层要求

  • 必须 使用 Figma 文本层，禁止将文字转为矢量或图片
  • 必须 行高、字间距使用明确数值，避免「Auto」导致浏览器与 Figma 计算不一致
  • 应当 段落宽度用 Auto Layout 的 Fixed / Fill 约束，而非手动拉文本框
  • 应当 长文案预设换行行为（truncate 场景单独标注 max-lines）
  • 必须 混排金额时：¥ 用 UI 字体，数字用 TCloudNumber，间距 4px

文字转曲、用 PNG 代替标题、用字符 < √ 代替图标
✓ 推荐
Text 图层 + 规范字号 + 矢量图标组件（chevron-left、check 等）

视觉铁律（高还原场景） 无阴影、无投影、无渐变。层级仅通过底色、1px 分割线、间距、字重区分——这类稿面导出最稳定。

效果建议

7组件与命名

组件化不仅为了设计效率，更是为了让插件能识别「这是按钮 / 标题栏 / 列表行」，输出语义化 HTML。

规范说明级别
使用 Component 实例PageTitleBar、Button、PageTabs 等须从组件库实例化，禁止每次重画必须
命名规范组件/变体/状态，如 Button/Primary/Default应当
图层命名英文或拼音+英文，避免「矩形 123」「Frame 456」应当
Variant 属性状态（Default/Hover/Disabled）用 Variant，不要复制多个独立组件应当
Detach 限制导出前检查是否误 Detach；Detached 实例无法关联组件规则必须

常用组件映射参考

Figma 组件期望 HTML 结构

Buttonbutton / a，文案 flex 居中
PageTabsnav + 等分 tab + 底部指示条
列表行flex row，左图标 + 中内容 + 右箭头

8图标与图片

图标

  • 必须 使用 SVG 矢量图标（组件库 Stratis / Iconex），禁止 emoji 占位
  • 必须 线性图标 stroke-width 统一为 2
  • 应当 图标作为 Component，颜色用 #333 / #929292 / #F32F41 规范色
  • 必须 返回箭头用 chevron-left，禁止文字 <

图片

  • 占位统一：浅灰底 #EDEDED + 居中白色 linear image 图标
  • 商品主图等比：1:1 或明确 aspect-ratio，用 Fill 而非拉伸
  • 导出格式：简单图标 SVG；照片 PNG/WebP @2x；避免超大未压缩图
  • 图片容器设置 clip / overflow hidden，防止溢出卡片圆角

9高风险项（导出前必查）

以下任一出现，还原度会显著下降
  1. 主体内容使用绝对定位（x/y 手工摆放）
  2. Group 代替 Frame，或 Auto Layout 未开启
  3. 间距 / 尺寸含小数（如 13.5px、7px）
  4. 隐藏层、锁定的废弃图层未删除
  5. 文本 OUTLINE 或 rasterize
  6. Tab 与卡片负间距层叠遮挡
  7. 白底卡片加 1px 描边（与 HTML 规范冲突时易 double border）
  8. 使用 Figma 特有效果：多重阴影、Inner shadow、Layer blur
  9. 布尔运算未 Flatten 的复杂矢量（部分插件解析失败）
  10. Frame 尺寸超出 390 导致横向溢出
插件无法替你「猜」意图 若设计稿是「看起来对」但结构混乱，导出 HTML 只会复现混乱的结构。 调整设计稿 = 为插件提供可机器读取的布局语义。

10验收标准

导出后建议按以下标准验收，而非凭感觉「差不多」。

验收项方法通过标准
整体布局 Figma 截图 vs 浏览器截图叠图 主要区块位置偏差 ≤ 2px
间距 DevTools 测量 padding / gap 与稿面标注一致，无累计漂移
字体 检查 computed font-family / size / line-height 字体文件已加载，字号行高一致
颜色 取色器对比 Hex 完全一致（注意导出是否转为 rgba）
文本换行 长文案、窄容器场景 换行位置与 Figma 一致或符合预设 truncate
响应 固定 390 宽视口 无横向滚动条，无元素裁切异常

推荐验收流程

  1. 选 1 个典型页（含标题栏 + 列表 + 按钮 + 卡片）做试点
  2. 导出 HTML 后在 390 宽视口打开
  3. 使用截图对比工具（Figma export vs 浏览器 full page screenshot）
  4. 未通过项回到 Figma 改结构，而非在 HTML 里手工 patch
  5. 试点通过后，同类页面批量导出
清单全部勾选 + 试点页验收通过 即可认为该批次设计稿已达到「可高保真导出」状态。后续维护时，新增页面仍须遵循本页规则。
